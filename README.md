# Brent Oil Price Change Point Analysis

Birhan Energies - Change Point Analysis and Statistical Modeling of Brent
crude oil prices (10 Academy Week 10 challenge).

## What this project does

1. **Task 1** (submitted separately): data pipeline, EDA, and a
   researched events dataset covering major geopolitical/economic events.
2. **Task 2**: a Bayesian single change point model (PyMC) that detects a
   structural break in Brent's daily log returns, and associates it with
   researched events.
3. **Task 3**: a Flask API + React dashboard that serves the analysis
   results and lets a stakeholder explore prices, the detected change
   point, and nearby events interactively.

## Task 2 result summary

A single change point model fit over the full 1987-2022 range does not
converge cleanly (r_hat > 1.1, hundreds of divergences, credible interval
spanning nearly the whole series) because the data contains several real
regime shifts (2008, 2014-2016, 2020). Scoping to **June 2019 - December
2020** gives a well-identified model:

- **Detected change point:** 2020-04-21 (one day after WTI futures went
  negative, 44 days after the Saudi Arabia-Russia price war began)
- **Average price before:** $57.20/barrel -> **after:** $40.42/barrel
  (-29.3%)
- **Mean daily log return:** -0.43% before -> +0.43% after

See `notebooks/02_change_point_model.ipynb` for the full walkthrough,
diagnostics, and discussion of why the full-range model doesn't identify
cleanly.

```bash
PYTHONPATH=. python run.py

# terminal 2
cd frontend
npm install
npm run dev
```

Then open http://localhost:5173.

## Assumptions and limitations

- A single change point model can only describe one structural break.
  The full history contains several; results here describe the model's
  best answer within the analyzed window, not a claim of one global break.
- Statistical association between a detected change point and a nearby
  event is not proof of causation - it is a plausible hypothesis that
  would need further corroboration (e.g. news volume, trading volume) to
  confirm.
- `tau`'s discreteness means its r_hat is naturally noisier than
  continuous parameters (PyMC samples it with Metropolis, not NUTS); a
  concentrated posterior histogram is the more informative diagnostic
  here than r_hat alone.
